### Draft 2: Architecture notes — DM to self

**Send to:** Personal notes

> Key architectural decisions worth remembering:
> - Demo should be a workspace, not a server mode. Restart-to-switch is a UX failure.
> - Daily plans are person-scoped, not workspace-scoped. One person, one plan per day across all roles. GitHub identity model, not Gmail fragmentation.
> - Port auto-detection (try preferred, increment if busy) is table stakes. Don't make users manage port conflicts.
> - Set the dependency budget before writing code, not after accumulating 40 packages.
